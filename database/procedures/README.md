# Guia de Instalação e Implementação: PL/Python no PostgreSQL

Este guia demonstra como configurar o suporte à linguagem Python dentro do PostgreSQL e como implementar uma lógica de transferência bancária com validação de saldo.

```SQL
drop table if exists accounts;

create table accounts (
    id int generated by default as identity,
    name varchar(100) not null,
    balance dec(15,2) not null,
    primary key(id)
);

insert into accounts(name,balance)
values('Bob',10000);

insert into accounts(name,balance)
values('Alice',10000);
```

A procedure `transfer` escrita em PL/PgSQL. Ela verifica se o remetente (sender) possui saldo suficiente antes de processar a transação.

## 1. Instalação do Suporte Python (PL/Python)

O **PL/Python** permite escrever procedures e funções utilizando o interpretador Python diretamente no SGBD.

### Passo 1: Instalação no Sistema Operacional

No terminal do seu servidor (Exemplo para Debian/Ubuntu), instale o pacote correspondente à sua versão do PostgreSQL (substitua `15` pela sua versão):

```bash
sudo apt-get update
sudo apt-get install postgresql-plpython3-15
```

### Passo 2: Habilitação no Banco de Dados

Acesse seu banco de dados como superusuário e execute o comando para criar a extensão:

```sql
CREATE EXTENSION IF NOT EXISTS plpython3u;
```

> Nota: O "u" indica untrusted, o que significa que a linguagem tem acesso a recursos do sistema operacional. Por segurança, apenas superusuários do banco podem criar funções nesta linguagem.

## 3. Como Testar

Para validar o funcionamento, você pode executar o seguinte comando SQL:

```SQL
CALL transfer_py(1, 2, 100.00);
```

### Verificação de Logs

Se o saldo for insuficiente, você verá uma mensagem no console:

> NOTICE: Transferência cancelada: Saldo insuficiente para o remetente 1.

## 4. Comandos Úteis

| Objetivo | Comando SQL |
| :--- | :--- |
| **Verificar linguagens instaladas** | `SELECT * FROM pg_language;` |
| **Remover a extensão** | `DROP EXTENSION plpython3u;` |
| **Testar versão do Python** | `CREATE FUNCTION py_ver() RETURNS text AS 'import sys; return sys.version' LANGUAGE plpython3u; SELECT py_ver();` |
| **Listar extensões ativas** | `SELECT * FROM pg_extension;` |

Ao contrário do PL/pgSQL, o Python dentro do Postgres não faz o "cast" (conversão) automático de tipos de forma tão flexível em alguns casos, então sempre use o plpy.prepare com os tipos de dados explícitos (como ["dec", "int"]) para evitar erros de compatibilidade!
